AUTHENTICATION FLOW:

1. User visits http://localhost:3000
   → Checks if already logged in
   → Redirects to /dashboard if authenticated
   → Shows landing page if not

2. User clicks "Sign In"
   → Navigates to /login
   → Sees login form with demo credentials

3. User enters credentials
   → System hashes password with bcrypt
   → Compares with stored hash
   → If valid, generates JWT tokens

4. System issues tokens:
   → Access Token (7 days)
   → Refresh Token (30 days)
   → Session created in database

5. Tokens stored:
   → Access token in HTTP-only cookie
   → Both tokens in localStorage
   → Session recovery enabled

6. User redirected to /dashboard
   → Dashboard loads with user data
   → User menu shows name and email
   → All API calls include Authorization header

7. User clicks "Sign Out"
   → API call to /api/auth/logout
   → Session deleted from database
   → Tokens cleared from storage
   → Redirect to /login

ENDPOINTS:

POST /api/auth/register
  Input:  { email, password, name, role }
  Output: { user, message }
  Roles:  EMPLOYEE, DEPARTMENT_HEAD, ASSET_MANAGER, ADMIN

POST /api/auth/login
  Input:  { email, password }
  Output: { user, accessToken, refreshToken }
  Auth:   None required

POST /api/auth/logout
  Input:  { refreshToken }
  Output: { message }
  Auth:   Bearer accessToken required

POST /api/auth/refresh
  Input:  { refreshToken }
  Output: { accessToken }
  Auth:   None required (uses refresh token)

All other endpoints:
  Auth:   Bearer accessToken required
  Header: Authorization: Bearer {accessToken}

PRODUCTION DEPLOYMENT:

Before deploying to production:

1. Generate secure JWT secrets:
   openssl rand -base64 32

2. Set environment variables:
   - Use Vercel Environment Secrets
   - Configure DATABASE_URL for production
   - Set NODE_ENV=production

3. Enable HTTPS:
   - Automatic on Vercel
   - Set secure: true in cookies

4. Database setup:
   - Create PostgreSQL database
   - Run migrations: npx prisma migrate deploy
   - Consider read replicas for scaling

5. Monitoring:
   - Set up error tracking
   - Monitor failed login attempts
   - Alert on security anomalies

6. Security:
   - Rotate JWT secrets monthly
   - Monitor for compromised tokens
   - Review audit logs regularly
