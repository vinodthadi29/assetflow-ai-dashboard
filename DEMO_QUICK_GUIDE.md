## VOICE-OVER TALKING POINTS

### Remember: Show Business Value, Not Features

**DON'T say**: "This page shows all assets in a table with sorting and filtering"
**DO say**: "Instantly find any asset across the entire enterprise"

**DON'T say**: "This is a conflict detection algorithm"
**DO say**: "The system prevents double-bookings before they happen"

**DON'T say**: "This is a machine learning model"
**DO say**: "AI predicts failures 90 days in advance, saving ₹500K per year"

### Key Phrases to Use
- "Autonomous intelligence"
- "Real-time visibility"
- "Prevents conflicts before they occur"
- "Predictive not reactive"
- "Enterprise-grade"
- "Production-ready"
- "Competitive advantage"
- "Business impact"
- "ROI-focused"

### Recording Tips
- Record each segment 2-3 times
- Don't record voice-over during screen recording (do it later)
- Move mouse deliberately and slowly
- Click clearly and precisely
- Pause 2 seconds at key moments
- Don't talk while recording screen (record voice-over separately)
- Keep segments <30 seconds for easier editing
- Have demo data pre-created (no creation during recording)

## COMMON MISTAKES TO AVOID

❌ **Too fast** - Demos that rush through features
→ Slow down, let features breathe

❌ **Too much jargon** - Technical deep-dives
→ Focus on business value, not implementation

❌ **Dead time** - Slow loading, thinking, searching
→ All actions should be instant (pre-load data)

❌ **Unclear voice-over** - Mumbling, filler words
→ Record multiple takes, use professional mic

❌ **Bad transitions** - Abrupt cuts, jarring changes
→ Use smooth 0.5s transitions

❌ **Showing errors** - Failed validations, conflicts
→ Show successful paths only (or show graceful error handling)

❌ **Generic data** - Placeholder text, fake values
→ Use realistic enterprise data
